Three PostgreSQL Architecture Constraints That Shape Scaling

🔧
Process-Per-Connection
Each client connection forks a new OS process consuming 5-10MB RAM
500 connections = 2.5-5GB overhead
📦
Row-Based Storage + MVCC
Full row copies on UPDATE. VACUUM consumes 10-30% CPU/IO. Reads all columns for analytics.
10-30x I/O amplification on analytics
💻
Single-Node Execution
No built-in distributed queries. Planner assumes all data is local. Limited parallel workers.
Cannot split joins across machines
